# Sales – ETL Data Engineering (Campus Use Case)
---

## 1. Executive Summary

This use case showcases a complete Informatica (ETL) data engineering mini-project using multiple datasets.

### Datasets Used (3–4 datasets)
- Customer_master.csv  
- Product_master.csv  
- Sales_transaction.csv  
- Sales_transaction_dirty.csv  

Students are required to read data from these CSV files and build a data pipeline to analyze the data and address the use cases described below.

---

## 2. Datasets (Only 3 Sources)

Files included in the pack:

### Source File Descriptions
1. **Customer_master.csv** – Contains customer details.
2. **Product_master.csv** – Contains product information.
3. **Sales_transaction.csv** – Clean sales transaction data.
4. **Sales_transaction_dirty.csv** – Contains incorrect/dirty sales records for data cleaning practice.

---

## Objective

Build an end-to-end ETL pipeline that:
- Reads data from multiple sources
- Cleans and validates data
- Handles dirty records
- Performs lookups using master data
- Loads processed data into the target system
- Supports analysis and reporting
